# Master PromptWars Submission Prompt

Use this as a high-stakes base prompt for coding rounds.

```text
Build the strongest possible solution for the given problem statement.

Primary judging priorities, in order:
1. Code quality
2. Security
3. Dynamic site behavior / interactivity
4. Problem statement alignment
5. Efficiency
6. Testing
7. Accessibility

Requirements:
- Follow the problem statement exactly.
- Write clean, modular, maintainable code with clear naming.
- Use secure defaults and avoid common vulnerabilities.
- For web tasks, make the site meaningfully dynamic and interactive.
- Keep the solution efficient and avoid unnecessary complexity.
- Include tests or a testing strategy with edge cases.
- Address accessibility where applicable.

Before finalizing, self-review for:
- code quality
- security
- dynamic behavior
- alignment to the task
- efficiency
- testing coverage
- accessibility
```

## Resubmission rule

If reviewing an existing submission with limited attempts remaining:

- recommend changes only when there is a genuine material improvement
- avoid rewriting for superficial wording changes
- prioritize gains in code quality, security, and dynamic behavior
- explicitly say `keep current submission` when confidence is low
